# مقترحات إضافات VS Code (VS Code Extensions RFCs)

يحوي هذا المجلد المقترحات (RFCs) المعتمَدة الخاصّة بـ**إضافات VS Code للغة ص**.

> الإضافات متعدّدة (قد تشمل إضافة اللغة الأساسية، وإضافة SadUI، وإضافات مرافِقة
> أخرى). أمّا أدوات اللغة (LSP، المنسّق…) فمقترحاتها في [`tools/`](../tools/)،
> واللغة نفسها في [`text/`](../text/).

## البنية: مجلّد فرعيّ لكلّ إضافة

لأنّ الإضافات متعدّدة، يُخصَّص لكل إضافة مجلّد فرعيّ خاصّ بها:

```
extensions/
├── sad-lang/       ← مقترحات إضافة لغة ص الأساسية
├── sad-ui/         ← مقترحات إضافة SadUI
└── .../            ← إضافات أخرى
```

## تسمية الملفات

- كل ملف: `NNNN-اسم-وصفي.md` حيث `NNNN` رقم الـ RFC = **رقم الـ PR في مستودع
  `sadlang-rfcs` نفسه** (لا في مستودع كود الإضافة).
- الترقيم **عامّ وفريد على مستوى المستودع كلّه** (لغة + أدوات + إضافات)، لأن مصدره
  مستودع مقترحات واحد — لا داخل كل مجلّد ولا حسب مستودع الكود.
- أثناء الـ Pull Request يبقى الاسم `0000-`؛ يُرقَّم عند الدمج.
- **حالة التنفيذ** تُتابَع عبر حقل `الحالة` في رأس الملف + Issue التتبُّع — راجع
  [README الجذر](../README.md#كيف-نعرف-هل-اكتمل-العمل-أم-ما-زال-مفتوحاً).

## العملية

نفس عملية المستودع الموحّدة في [README الجذر](../README.md): انسخ القالب
[`0000-template.md`](../0000-template.md)، املأه، افتح PR. وجود مقترح هنا يعني
الموافقة على **مبدئه** لا اكتمال تنفيذه — راجع Issue التتبُّع في مستودع الإضافة المعنيّ.

> **أمان الدمج** (CI، البوّابات، العقود) يعيش في **مستودع كود الإضافة نفسه** ويُصمَّم
> وفق داخليّاته — هذا المستودع طبقة المقترحات (RFC) الموحّدة فقط.
