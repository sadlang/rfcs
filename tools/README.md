# مقترحات الأدوات (Tooling RFCs)

يحوي هذا المجلد المقترحات (RFCs) المعتمَدة الخاصّة بـ**أدوات لغة ص الرسمية** —
مثل خادم LSP، والمنسّق (Formatter)، وأداة `sadinfo`، ومدير الحزم، وغيرها.

> **اللغة نفسها** مقترحاتها في [`text/`](../text/). **إضافات VS Code**
> مقترحاتها في [`extensions/`](../extensions/). هذا المجلد للأدوات بين بين.

## البنية: مجلّد فرعيّ لكلّ أداة

لأنّ الأدوات متعدّدة، يُخصَّص لكل أداة مجلّد فرعيّ خاصّ بها، فلا تختلط مقترحاتها:

```
tools/
├── lsp/            ← مقترحات خادم LSP
├── formatter/      ← مقترحات المنسّق
├── sadinfo/        ← مقترحات أداة sadinfo
└── package-manager/← مقترحات مدير الحزم
```

## تسمية الملفات

- كل ملف: `NNNN-اسم-وصفي.md` حيث `NNNN` رقم الـ RFC = **رقم الـ PR في مستودع
  `sadlang-rfcs` نفسه** (لا في مستودع كود الأداة).
- الترقيم **عامّ وفريد على مستوى المستودع كلّه** (لغة + أدوات + إضافات)، لأن مصدره
  مستودع مقترحات واحد — لا داخل كل مجلّد ولا حسب مستودع الكود.
- أثناء الـ Pull Request يبقى الاسم `0000-`؛ يُرقَّم عند الدمج.
- **حالة التنفيذ** تُتابَع عبر حقل `الحالة` في رأس الملف + Issue التتبُّع — راجع
  [README الجذر](../README.md#كيف-نعرف-هل-اكتمل-العمل-أم-ما-زال-مفتوحاً).

## العملية

نفس عملية المستودع الموحّدة الموضّحة في [README الجذر](../README.md): انسخ القالب
[`0000-template.md`](../0000-template.md)، املأه، افتح PR. وجود مقترح هنا يعني
الموافقة على **مبدئه**، لا اكتمال تنفيذه — راجع Issue التتبُّع في مستودع الأداة المعنيّ.

> **أمان الدمج** (اختبارات منع التراجع، البوّابات، العقود) يعيش في **مستودع كود
> الأداة نفسه**، ويُصمَّم وفق داخليّاته — هذا المستودع طبقة المقترحات (RFC) فقط.
